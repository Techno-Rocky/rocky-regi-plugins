# 売上レビュー(方向で読む)

## これは何か
店長が「今週どうだった?」と聞いたときに、**金額を出して終わりにしない**。数字を「方向(上向き/下向き)」に翻訳し、それが一時的か連続しているかを確かめ、要因の見当をつけ、次に何を見るべきかまで導く。ロッキーレジの分析の起点であり、着地点。

## 使うMCPツール(rocky-regi)
- `get_business_summary` — 当期の売上/客数/客単価と比較期(compare_from/compare_to)の差分。**このフローの主軸**。
- `get_store_insights` — 店の自動気づき(severity = warning / positive / info、target = abc/engineering/heatmap/basket/loss/visit-source)。
- `get_goals` / `get_goal_progress` — 目標がある店の着地見込み。
- `show_today_summary` / `show_sales_dashboard` — リッチUIで描画。

## 手順(この順で多段に呼ぶ)

### 1. 期間を決める
発話から期間を決める。明示がなければ **当営業日** を既定にし、加えて **直近7日** も併算して「今日単発」と「週トレンド」の両方を出す。「今週」なら当週 vs 前週、「今月」なら当月 vs 前月。

### 2. 当期 vs 前同期を取る(方向を出す)
`get_business_summary(from, to, compare_from, compare_to)` を呼ぶ。返ってきた差分から:
- **方向**: 売上が前同期比で 上向き か 下向き か
- **内訳**: その変化が「客数」由来か「客単価」由来かを分解(客数減×客単価増 なら"少人数高単価にシフト"など、意味が変わる)

### 3. 下向きのときだけ連続性を確かめる(遅延評価)
方向が **下向きのときだけ**、`get_business_summary` を直近3区間ぶん反復して「N週連続で下向きか」を自分で判定する。
- 上向きなら追撃しない(無駄な呼び出しをしない)。
- rockyにはトレンド列が無いので、区間を並べて自分で連続性を組み立てる。これがフロー化の価値。
- **注意**: `get_business_summary` はレート制限(5回/分・100回/日)がある。反復は3区間までに抑える。

### 4. 気づきを裏付けに紐づける
`get_store_insights(from, to)` を呼び、`severity = warning` の気づきを方向の裏付けとして最大2件まで拾う。target(engineering/loss/heatmap/basket)は「次にどのツールを見ればいいか」を指すので、深掘りの入口として覚えておく。

### 5. 目標がある店だけ着地見込みを添える
`get_goals` で進行中ゴールがあれば `get_goal_progress` で達成率を取る。**経過日数比の単純ペース按分**で着地見込みを方向で表現する(例: 月の6割経過で達成率5割なら"このペースだと未達方向")。
- `unmet_reason` が「データ不足/k匿名下限未満/未確定」を示すなら、**「まだ判定できない」と正直に返す**。達成率が低い=未達、と早合点しない。

### 6. 欠損の正しい扱い(重要)
- インサイトがゼロ = 「問題なし」ではない。**「検知ルールに該当しなかった」**と表現する。
- 原価(`cost_excl_tax`)未設定の店は粗利系インサイトが出ない。その場合は**「粗利は判定不能」**と添える。数字が無いことを"良い"と誤読させない。

## 出力の型(表現ゆれを防ぐため固定)
会話の締めは必ずこの並びで:

```
[方向] 前週比 [±%](客数[±]/客単価[±円])
連続: [N]週 [上向き/下向き]
要因候補: [インサイト最大2件、無ければ「特記なし(ルール非該当)」]
次に見る: [死に筋なら「死に筋追跡」、急落なら「原因究明」など次フロー]
```

そのうえで `show_today_summary` または `show_sales_dashboard` でUIを描画する。

## やってはいけないこと(ガード)
- **金額だけ出して終わらない**。方向・連続性・次の一手まで必ず出す(それがこのスキルの存在理由)。
- インサイトゼロを「好調」と言い換えない。
- 目標未達を、データ不足と区別せずに断じない。
- レート制限を超える反復呼び出しをしない(下向きのときだけ3区間)。
- このスキルは**読み取りのみ**。売上を変える操作はしない。打ち手は「次にこう動く」の提案に留め、実行系(プロモ作成等)は別スキル(季節プロモ差し替え)に渡す。

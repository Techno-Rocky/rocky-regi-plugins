# 店舗クリエイティブ画像生成(用途から公開まで一本で)

## これは何か
店長が「LINEに載せるバナーが欲しい」「新メニューの写真を作って」「ロゴを更新したい」と言ったときに、生成AIで画像を出して渡すだけで終わらせない。用途と方向性を先に固め、候補を複数出して選ばせ、EXIFやmXSSの安全ゲートを通し、薬機法・景表法・食品表示のNG訴求を人間レビューに回し、最後は必ず人間publishで公開する。ロッキーレジ側は「受け皿」に徹し、知能は外部AI(nano banana / Gemini画像)側に置く。#1672のクリエイティブ制作パート。

## 使うMCPツール(rocky-regi)
- get_store_context — 店の業態/客層/雰囲気/既存トーンを取る。プロンプト生成の下敷き。
- get_store_profile — 店名/ブランドカラー/既存ロゴなど、生成の一貫性を保つための素材。
- create_line_message_asset_upload_url — LINE配信素材の署名付きPUT URLを発行。アップロード先はここ経由に限定。
- update_store_logo — 店舗ロゴの差し替え。人間承認後にのみ叩く。
- get_line_messaging_quota — LINE配信素材として使う前に、そもそも今月の送信枠が残っているかを確認(枠がないなら制作を急がない)。

## 外部ツール(rocky-ops連結)
- nano banana / Gemini画像 — 画像生成本体。プロンプト→複数候補。
- Claude/Anthropic API — キャプション/コピー案の生成が必要なとき。
- Meta公式Ads MCP — 未接続。広告配信への直接投入は本スキルの範囲外。

## 手順(この順で多段に呼ぶ)
### 1. 用途と方向性を確認する
発話から次を必ず言語化してから生成に入る。曖昧なまま生成しない。
- 用途 — LINE配信素材 / メニュー写真 / 店舗ロゴ / F10店舗サイトのバナー・キービジュアル、のどれか。
- 方向性 — 季節(秋/冬/母の日等)、雰囲気(落ち着き/賑わい/高級感)、対象客層(常連/新規/ファミリー/若年)。
- 制約 — 縦横比、テキスト有無、ブランドカラー、既存ロゴとの整合。
用途で「取り込み先のrocky側ツール」が決まる(LINE→create_line_message_asset_upload_url、ロゴ→update_store_logo、メニュー写真/サイト画像→メディアギャラリー経由)。ここが決まらないとアップロード先も決まらないので、必ず先に確定させる。

### 2. get_store_context / get_store_profileでトーンを取る
店の業態・既存ブランドカラー・既存ロゴ・既存写真のトーンを取り、プロンプトに織り込む。取れない項目は無理に埋めず「未取得」として生成側に伝える(勝手にブランド像を捏造しない)。

### 3. 外部AI(nano banana / Gemini画像)で候補を複数出す
1枚で決めない。最低3案、方向性の振れ幅を持たせて出す(例: 落ち着き寄り / 賑わい寄り / 中間)。プロンプトはユーザーに提示可能な形で残す(あとで微修正できるように)。

### 4. 候補を提示して人間に選ばせる
生成した候補は必ず人間に見せて選ばせる。AIが独断で1枚を「これで行きます」と決めない。ボツ理由が出たら、方向性を再修正して 3. に戻る。

### 5. 取り込み前の安全ゲート(自動)
選ばれた画像を rocky に上げる前に、次を必ず自動で通す。1つでも引っかかったら取り込みを止め、人間に報告する。
- EXIF除去 — 位置情報(GPS)や端末情報を必ず落とす。飲食店の場合、店舗以外で撮った素材や生成AIメタデータが乗るとリスク。
- mXSS / 悪意SVG検査 — SVGを扱うときは script/onload/foreignObject等の危険要素を除去。通らないものは弾く。
- サイズ・形式 — 用途別の上限(LINE素材/ロゴ/バナー)に収まるか確認、超えたら再生成 or 縮小。
- 画像フォーマット — 想定外の拡張子(実体不一致)は弾く。

### 6. 表示規制チェック(薬機法/景表法/食品表示)
画像内テキストや、同時に生成するキャプション案について、次のNG訴求を必ずチェックする。1つでも該当したら人間レビューへ回し、AIだけで通さない。
- 薬機法 — 「効く」「治る」「痩せる」等の効能効果の断定表現。健康食品/飲料で頻発。
- 景表法 — 「日本一」「絶対」「業界最安」等の合理的根拠のない優良誤認/有利誤認。
- 食品表示 — 原材料・原産地・アレルゲンと画像や文言が乖離していないか(実物と違うトッピングを描いていないか等)。
- 生成AIの権利表示 — 生成物であること、権利者要件(モデル素材/実在人物ではないか)を確認。

### 7. 取り込み(用途別のrocky受け皿へ)
安全ゲートと表示規制を通ったものだけを、用途別のrocky側APIで取り込む。
- LINE配信素材 → create_line_message_asset_upload_url で署名付きPUT URLを発行 → その URL に対して外部からアップロード。rocky を経由しない直アップロードは禁止。
- 店舗ロゴ → update_store_logo。既存ロゴとの差分を人間に必ず提示。
- メニュー写真/F10サイト画像 → メディアギャラリー経由(この段階は本スキルの範囲外、別スキル/管理画面へ渡す)。

### 8. 人間publish承認(自動publish禁止)
アップロード完了=公開ではない。必ず人間の publish 操作を最後に挟む。AIが独断で「公開しました」に進めない。LINE配信の実送信は本スキルの範囲外(LINE配信スキルに渡す)。

## 出力の型(表現ゆれを防ぐため固定)
```
用途: <LINE配信素材 / 店舗ロゴ / メニュー写真 / F10店舗サイト画像>
方向性: <季節・雰囲気・対象客層を1行>
候補: <3案の要約と、それぞれの狙い>
選定結果: <人間が選んだ案 / 差し戻し理由>
安全ゲート: EXIF除去=OK / mXSS=OK / サイズ=OK(<横×縦, KB>)
表示規制チェック: 薬機=OK/要確認 / 景表=OK/要確認 / 食品表示=OK/要確認
取り込み先: <create_line_message_asset_upload_url / update_store_logo / メディアギャラリー>
アップロード状態: <未実行 / 完了、まだ非公開>
publish承認: <未取得 / 取得済み(承認者と日時)>
次アクション: <人間publish待ち / 表示規制の再確認 / 再生成>
```
UIウィジェットで確認したいときは、店舗コンテキストは show_store_profile、LINE配信素材として使うなら残枠を get_line_messaging_quota で添える。

## やってはいけないこと(ガード)
- AIが独断で1枚に決めない — 必ず複数候補+人間選択。
- 安全ゲートをスキップしない — EXIF位置情報の残留、mXSS/悪意SVGは自動除去/自動弾き。1件でも通ったら公開しない。
- 薬機/景表/食品表示のNG訴求を、AIだけで「セーフ」と判定しない — 疑わしきは必ず人間レビュー。
- 実物と乖離した表現をしない — 提供していないトッピング/食材/量を画像で誇張しない。景表法/食品表示リスクに直結する。
- 生成物であることを隠さない — 実在人物風・他社ブランド類似は避ける。権利表示を怠らない。
- 自動publishしない — アップロード完了と公開は別。最後は必ず人間publish。
- rockyを経由しない直アップロード禁止 — LINE素材は必ず create_line_message_asset_upload_url の署名付きPUT経由。
- 「レジが自動で画像を作って配信する」等の誇張表現をしない — 画像生成は外部AI、rockyは受け皿。
- 未接続の外部ツール(Meta公式Ads MCP)へ流し込まない — 現状は本スキルの範囲外。
- #1674のポータル巡回(Claude in Chrome経由の外部サイト自動操作)はBOT規約リスクで保留中。本スキルからは呼ばない。
- レート制限を超える反復生成をしない — ボツが続くときは方向性の言語化に戻る(むやみに再生成を叩かない)。
- 読み取り/生成のみで完結できる範囲を超えたら、必ず別スキル(LINE配信 / メニュー編集 / F10店舗サイト公開)へ渡す。

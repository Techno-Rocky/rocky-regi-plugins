# rocky-regi プラグイン

ロッキーレジ（飲食店POS）を **claude.ai から自然言語で回す**ための業務フロープラグイン集です。
「今週どうだった?」「この売れないメニュー、どう直せばいい?」と話しかけるだけで、Claude がロッキーレジのデータを読み、方向を判断し、打ち手まで導きます。

## 思想

**知能は外側（Claude）、レジはデータ供給に徹する。** ロッキーレジ本体にAIは載せません。決定論的な数字はレジのMCPが返し、その解釈・優先順位付け・打ち手への翻訳・外部への発信は、外側のAI（このプラグインのスキル）が担います。だから各スキルは「ツール1回で数字を出す」単発ではなく、**複数の機能を繋いで変化を読み、打ち手まで導く多段フロー**です。

## プラグイン構成（立場別）

| プラグイン | 対象 | 中身 |
|---|---|---|
| **rocky-store-manager** | 店長・単店運営 | 売上を方向で読む／死に筋を追跡／季節プロモ差し替え／急落の原因究明 |
| **rocky-hq** | 経営者・本部 | 全店を横串で俯瞰／外れ値店の発見／施策の効き検証と横展開／離反客の掘り起こし |
| **rocky-ops** | 全職責横断 | 合成消費者調査(SSR)・画像生成・広告運用と、それらを内向き分析に繋ぐ**連結フロー** |
| **rocky-floor** | 現場スタッフ | 営業中の卓・注文・在庫売切など瞬時の操作（第二弾以降で拡充） |

権限は claude.ai ログイン（Clerk OAuth）で自動判定されます。現場スタッフが経営操作を呼んでも、レジ側が権限で弾くので安全です。

## 導入手順（店舗オーナー向け）

1. **claude.ai** を開き、左サイドバー → **Customize（カスタマイズ）** → **Plugins（プラグイン）** タブ
2. **＋** → マーケットプレイスの追加で、このリポジトリの URL を指定
   ```
   https://github.com/Techno-Rocky/rocky-regi-plugins
   ```
3. 使いたいプラグイン（例: `rocky-store-manager`）を **Install**
4. 初回にロッキーレジへの接続認可（ブラウザで自分のロッキーレジ・アカウントにログイン）
5. 会話で「今週どうだった?」などと話しかける

> 接続先: `https://regi.techno-rocky.com/api/v1/mcp`（認証は各自のログインで行われるため、鍵の入力は不要です）

## 外部ツールについて（rocky-ops）

`rocky-ops` の一部フロー（合成消費者調査・画像生成・広告）は、ロッキーレジの外側のAI・サービスを使います。

- **合成消費者調査(SSR)・画像生成**: Claude/生成AIで実行。**予言ではなく参考シナリオ**として扱います（誇張しません）。
- **Meta広告運用**: Meta公式のツール接続が別途必要です。未接続でも、内向きの分析だけで完結するように作ってあります。
- **外部ポータル更新**: 各社の規約確認が済むまで有効化しません。

## 注意

- 売上や在庫を**変更する操作**（プロモ作成・在庫売切など）は、必ず内容を提示して確認を取ってから、1操作ずつ実行します。全店一括反映はしません。
- 効果測定は「見かけの数字」ではなく、可能な限り lift（因果の増分）で誠実に扱います。
- このプラグインは配布用の「入口」です。実際のデータはロッキーレジの認証で守られています。
